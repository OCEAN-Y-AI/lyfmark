# Öffentliche Dokumentation (Kunde/Redaktion)

Diese Dokumentation ist für nicht-technische Redakteure und Entscheider gedacht.

## Start in 2 Klicks

1. Onboarding: `docs/public/onboarding.md`
2. Danach je nach Aufgabe:
	- Installation starten: `docs/public/installation.md`
	- Inhalte schreiben: `docs/public/content-richtlinien.md`
	- Menü pflegen: `docs/public/menu.md`
	- Template wechseln: `docs/public/templates.md`
	- Module auswählen: `docs/public/modules/README.md`
	- Beispiele kopieren: `docs/public/examples/README.md`
	- Änderungen im Überblick: `docs/public/changelog.md`
	- Git-Spickzettel: `docs/public/github-git-cheat-sheet.pdf`

## Hinweis

Interne Technik-, Architektur- und Produktdokumentation liegt bewusst getrennt unter `docs/internal/`.
