# contact-form Testcases

## Core

- A minimal valid `:::contact-form` block builds successfully and renders semantic HTML.
- Invalid module content fails with a clear, actionable build error message.
- Rendered output for `contact-form` does not contain raw `:::` directive markers.

## Attributes

- `heading`: valid values are accepted and invalid values fail fast with a clear message.
- `style`: valid values are accepted and invalid values fail fast with a clear message.
- `subheading`: valid values are accepted and invalid values fail fast with a clear message.
- `submit`: valid values are accepted and invalid values fail fast with a clear message.
