# LyfMark Testcase Index

## Scope Split Rule

- `src/lyfmark/testcases/global.md` contains only cross-module and system-wide contracts.
- `src/lyfmark/modules/<module>/testcases.md` contains all module-specific behavior and regressions.
- If a testcase mentions one concrete directive/module as primary subject, it belongs to that module file.
- If a testcase spans parser, formatter, build, menu, extension, config, or multiple modules at once, it belongs to `global.md`.
- Keep module files concise and actionable for AI and human developers working on exactly one module.
